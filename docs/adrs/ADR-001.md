# Architectural Decision Record (ADR) - 001
## Stack Tecnológica: Berserkr VTT

### Contexto
Necessidade de um sistema robusto, fácil de manter e que facilite o aprendizado de novas tecnologias (Svelte) para o Foundry VTT.

### Decisão
Utilizaremos:
1. **TypeScript**: Para tipagem estática e segurança na API do Foundry.
2. **Svelte**: Para a interface de usuário (fichas e diálogos) devido à sua reatividade superior ao Handlebars.
3. **Data Models (v10+)**: Para definição de esquemas de dados de Atores e Itens.
4. **Vite**: Como bundler para um ciclo de desenvolvimento rápido (HMR).
5. **Versão Alvo (V13)**: O desenvolvimento é focado na API da Versão 13, utilizando tipos e padrões modernos, mantendo compatibilidade retroativa com a Versão 12 através de detecção de funcionalidades (feature detection).

### Consequências
- **Positivas**: Facilidade de refatoração, reatividade instantânea na UI, código mais legível e preparado para o futuro do Foundry.
- **Negativas**: Necessidade de manter lógica de "fallback" (como detecção de classes em `foundry.applications`) para garantir que usuários na V12 não tenham o sistema quebrado.
