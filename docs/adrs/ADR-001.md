# Architectural Decision Record (ADR) - 001
## Stack Tecnológica: Berserkr VTT

### Contexto
Necessidade de um sistema robusto, fácil de manter e que facilite o aprendizado de novas tecnologias (Svelte) para o Foundry VTT.

### Decisão
Utilizaremos:
1. **TypeScript**: Para tipagem estática e segurança na API do Foundry.
2. **Svelte**: Para a interface de usuário (fichas e diálogos) devido à sua reatividade superior ao Handlebars.
3. **Data Models (v10+)**: Para definição de esquemas de dados de Atores e Itens.
4. **Vite**: Como bundler para um ciclo de desenvolvimento rápido (HMR).

### Consequências
- **Positivas**: Facilidade de refatoração, reatividade instantânea na UI, código mais legível.
- **Negativas**: Complexidade inicial de configuração do ambiente de build para o Foundry.
