# ADR-003: Fluxo de Trabalho e Mandatos de IA

## Contexto
Necessidade de padronizar o fluxo de desenvolvimento, ferramentas e comunicação entre o desenvolvedor e a IA.

## Decisão
Estabelecemos as seguintes regras de operação:
1. **Fonte da Verdade**: `docs/TASKS.md` é a referência absoluta para o progresso do projeto.
2. **Docker**: Todos os comandos de construção, instalação e execução (ex: `npm install`) devem ser realizados via Docker.
3. **Idioma**: Código em Inglês, comentários em Português (pt-br) explicando a lógica.
4. **Normalização**: Campos de texto normalizados devem ser em lowercase, removendo acentos (ex: `Str::lower(Str::ascii($value))`).
5. **Testes**: Novas funcionalidades de lógica devem possuir testes unitários (Vitest).

## Consequências
- Consistência total entre as sessões da IA.
- Ambiente de desenvolvimento isolado e reprodutível via Docker.
- Código legível globalmente com explicações locais em português.
